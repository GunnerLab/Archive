Job Scheduler

To run jobs that may take hours or days to finish, you need a job scheduler so that the jobs may be distributed over the slave nodes. These jobs must be able to run silently at the background. The job scheduler on Hestia is condor (http://www.cs.wisc.edu/condor/).

To submit a job or a batch of jobs, you need to prepare a condor submit file. In this file you specify the program, directories the program will run, and the file you record the output of the program.

Here is an example of a submit file named as "submit":

Executable = /home/mcce/mcce1.0/mcce
#Arguments = 
Universe = vanilla
error   = condor.err
Log     = condor.log
output = run.log
Notification = never

Initialdir = 1a2p_a/
queue

Initialdir = 1b2x_a/
queue

To submit jobs, run:

condor_submit submit

Job scheduler will take care of the job queue, balance the load, monitor the progress and inform the users when jobs are done. For summary of condor command, refer to pdf file condor_manual.pdf.

If you have urgent jobs, ask the system administrator to set higher priority for your jobs.
