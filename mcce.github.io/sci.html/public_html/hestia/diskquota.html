Disk Quota and Backup on Hestia

From the past experience on cluster metis and sibyl, the disk is used up quickly. To avoid system crash and loss of important big calculation, Hestia is set up a "soft" disk quota mechanism, with which an email will be sent to the user who exceeds the quota, 30 GB, or to all users if the disk is more than 90% full.

Users are responsible for their own data backup. The system administor will only backup MCCE code and system configuration files.

Examples of backup:

To backup directory /home/jmao/rxn" on Hestia to my local computer:

On Hestia,

tar -zcvf rxn.tar.gz rxn

On my local computer,

scp jmao@134.74.186.110:./rxn .

To restore the directory:

On my local computer,

scp rxn.tar.gz jmao@134.74.186.110:./

On Hestia,

tar -zxvf rxn.tar.gz
